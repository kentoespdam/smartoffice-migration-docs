# 5.3 Recipients

## `GET /v1/mails/{mailId}/recipients`

List semua recipient surat.

**Response `200 OK`:**
```json
{
  "success": true,
  "data": [
    {
      "id": 100,
      "userId": 20,
      "empId": 789,
      "empCode": "EMP001",
      "empName": "Andi Pratama",
      "posId": 50,
      "posName": "Kasi SDM",
      "circulation": "TO",
      "circulationName": "Kepada",
      "emailNotif": true,
      "smsNotif": false
    }
  ]
}
```

---

## `POST /v1/mails/{mailId}/recipients`

Tambah recipient ke surat (harus status DRAFT).

**Request Body:**
```json
{
  "empId": 789,
  "circulation": "TO"
}
```

**Field Notes:**
- `empId`: ID karyawan. Server akan lookup `user_id`, `emp_name`, `pos_id`, `pos_name` dari HR Service / local cache.
- `circulation`: `"TO"` | `"CC"` | `"DISPOSITION"` | `"INFO"`

**Response `201 Created`:**
```json
{
  "success": true,
  "data": {
    "id": 101,
    "userId": 20,
    "empId": 789,
    "empName": "Andi Pratama",
    "posId": 50,
    "posName": "Kasi SDM",
    "circulation": "TO"
  }
}
```

**Error `409 Conflict`:**
```json
{
  "success": false,
  "errorCode": "RECIPIENT_DUPLICATE",
  "message": "Karyawan ini sudah terdaftar dalam list penerima"
}
```

**Error `422`:**
```json
{
  "success": false,
  "errorCode": "RECIPIENT_NO_USER_ACCOUNT",
  "message": "Karyawan tidak memiliki user aplikasi"
}
```

---

## `PATCH /v1/mails/{mailId}/recipients/{recipientId}`

Update notification flag recipient.

**Request Body:**
```json
{
  "emailNotif": true,
  "smsNotif": false
}
```

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Recipient updated"
}
```

---

## `DELETE /v1/mails/{mailId}/recipients/{recipientId}`

Hapus recipient dari surat (harus status DRAFT).

**Response `204 No Content`**
