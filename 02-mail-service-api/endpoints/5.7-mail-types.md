# 5.7 Lookup: Mail Types

## `GET /v1/mail-types`

List jenis surat aktif.

**Query Parameters:**

| Param | Type | Default |
|---|---|---|
| `page` | int | 1 |
| `size` | int | 50 |

**Response `200 OK`:**
```json
{
  "success": true,
  "data": [
    { "id": 1, "name": "Internal", "status": "ACTIVE" },
    { "id": 2, "name": "Masuk", "status": "ACTIVE" },
    { "id": 3, "name": "Keluar", "status": "ACTIVE" }
  ]
}
```

---

## `GET /v1/mail-types/tree`

Mail types sebagai tree (untuk sidebar/nav). Dipakai di frontend untuk filter arsip dan surat.

**Response `200 OK`:**
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "name": "Internal",
      "iconClass": "email-open-image"
    }
  ]
}
```

---

## `POST /v1/mail-types`

**Role**: MAIL_ADMIN

**Request Body:**
```json
{
  "name": "Surat Keputusan"
}
```

**Response `201 Created`:**
```json
{
  "success": true,
  "message": "Data berhasil disimpan",
  "data": { "id": 4, "name": "Surat Keputusan" }
}
```

**Error `409`:** Duplikasi nama.

---

## `PUT /v1/mail-types/{typeId}`

**Role**: MAIL_ADMIN

**Request Body:**
```json
{
  "name": "Surat Keputusan (SK)"
}
```

---

## `DELETE /v1/mail-types/{typeId}`

**Role**: MAIL_ADMIN. Gagal jika masih ada kategori aktif yang menginduk.

**Error `422`:**
```json
{
  "success": false,
  "errorCode": "TYPE_HAS_CATEGORIES",
  "message": "Terdapat Kategori Surat yang menginduk ke Jenis Surat ini"
}
```
