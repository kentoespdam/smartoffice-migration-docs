# 5.2 Mails

## `GET /v1/mails`

List surat dalam folder tertentu (paginated). **Wajib** kirim `folderId`.

**Query Parameters:**

| Param | Type | Required | Default | Keterangan |
|---|---|---|---|---|
| `folderId` | int | ‚úÖ | ‚Äî | ID folder |
| `keyword` | string | ‚Äî | ‚Äî | Search: subject, content, note, sender, recipient, no surat, category, type |
| `startDate` | date | ‚Äî | ‚Äî | Filter tanggal surat (m_date) mulai |
| `endDate` | date | ‚Äî | ‚Äî | Filter tanggal surat (m_date) sampai. Jika ada date range, surat unread tetap ditampilkan |
| `page` | int | ‚Äî | 1 | |
| `size` | int | ‚Äî | 20 | |
| `sort` | string | ‚Äî | `createdDate,desc` | Sortable fields: `createdDate`, `date`, `subject`, `mailNumber`, `type`, `circulation` |

## `POST /v1/mails`

Buat surat baru (draft). Bisa langsung kirim jika `send=true`.

**Request Body:**
```json
{
  "mailNumber": "",
  "date": "2026-03-12",
  "typeId": 1,
  "categoryId": 86,
  "subject": "Undangan Rapat Koordinasi",
  "content": "<p>Dengan hormat...</p>",
  "note": "Harap membawa laptop",
  "maxResponseDate": "2026-03-15",
  "rootId": 0,
  "parentId": 0,
  "incomingMail": {
    "number": "",
    "origin": "",
    "date": null
  },
  "outgoingMail": {
    "destination": "",
    "recipient": ""
  },
  "send": false
}
```

**Field Notes:**
- `mailNumber`: Kosong `""` atau `"[Nomor Otomatis]"` ‚Üí nomor digenerate otomatis saat send. Jika diisi manual, dipakai apa adanya.
- `rootId` & `parentId`: `0` untuk surat baru. Untuk reply/disposisi, isi dengan ID surat yang dibalas.
- `send`: `true` ‚Üí langsung kirim setelah create (skip draft). `false` ‚Üí simpan sebagai draft.

**Response `201 Created`:**
```json
{
  "success": true,
  "message": "Surat berhasil disimpan",
  "data": {
    "id": 180002,
    "status": "DRAFT"
  }
}
```

**Response `201 Created` (send=true):**
```json
{
  "success": true,
  "message": "Surat berhasil dikirim",
  "data": {
    "id": 180002,
    "mailNumber": "001/I/HRD-485.1/III/2026",
    "status": "SENT"
  }
}
```

**Error `422` (send tanpa recipient):**
```json
{
  "success": false,
  "errorCode": "MAIL_NO_RECIPIENT",
  "message": "Surat gagal dikirim. Daftar penerima belum ada."
}
```

## `PUT /v1/mails/{mailId}`

Update draft surat. Hanya bisa jika `status=DRAFT` dan user adalah creator.

**Request Body:** Sama dengan POST (tanpa `rootId` dan `parentId`).

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Surat berhasil diperbaharui",
  "data": {
    "id": 180002,
    "status": "DRAFT"
  }
}
```

## `POST /v1/mails/{mailId}/reply`

Buat surat balasan / disposisi terhadap surat ini. Otomatis set `rootId` dan `parentId`, serta copy recipients dari thread.

**Request Body:**
```json
{
  "date": "2026-03-13",
  "typeId": 1,
  "categoryId": 86,
  "subject": "Re: Undangan Rapat Koordinasi",
  "content": "<p>Baik, akan hadir...</p>",
  "note": "",
  "maxResponseDate": "2026-03-15",
  "copyRecipients": "THREAD",
  "send": true
}
```

**Field Notes:**
- `copyRecipients`:
  - `"PARENT"` ‚Üí Copy recipients dari parent mail saja (exclude self)
  - `"THREAD"` ‚Üí Copy ALL unique recipients dari seluruh thread + root creator (exclude self)
  - `"NONE"` ‚Üí Tidak copy, user tambahkan manual

**Response `201 Created`:**
```json
{
  "success": true,
  "message": "Surat berhasil dikirim",
  "data": {
    "id": 180003,
    "mailNumber": "002/I/HRD-485.1/III/2026",
    "status": "SENT",
    "recipientsCopied": 3
  }
}
```

**Side Effects:**
- Parent mail dipindahkan ke folder READ_ITEMS untuk current user (jika reply dari INBOX)
- Response time tracking record diinsert ke `mail_respontime` (jika parent creator ada di recipient list)

## `POST /v1/mails/{mailId}/move`

Pindahkan surat ke folder lain.

**Request Body:**
```json
{
  "targetFolderId": 15
}
```

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Surat berhasil dipindahkan"
}
```

---

## `POST /v1/mails/{mailId}/mark-read`

Pindahkan surat dari current folder ke READ_ITEMS dan set read_status=READ.

**Request Body:** (kosong)

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Surat dipindahkan ke Folder [Read Items]"
}
```

---

## `DELETE /v1/mails/{mailId}`

Soft-delete surat ke Trash. Jika sudah di Trash ‚Üí permanent hide (folder_id = -1).

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Surat berhasil dihapus"
}
```

---

## `POST /v1/mails/{mailId}/restore`

Restore surat dari Trash ke folder asalnya.

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Surat berhasil di-restore"
}
```

---

## `DELETE /v1/mails/trash`

Kosongkan seluruh isi Trash folder (permanent hide).

**Response `200 OK`:**
```json
{
  "success": true,
  "message": "Folder berhasil dikosongkan"
}
```
