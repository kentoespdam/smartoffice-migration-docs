# 5.9 Attachments

## `POST /v1/mails/{mailId}/attachments`

Upload attachment ke surat. Multipart form-data.

**Request:** `Content-Type: multipart/form-data`

| Field | Type | Keterangan |
|---|---|---|
| `file` | binary | File to upload |
| `notes` | string | Keterangan opsional |

**Response `201 Created`:**
```json
{
  "success": true,
  "data": {
    "id": 501,
    "fileName": "lampiran.pdf",
    "fileExt": "pdf",
    "fileSize": 124000,
    "notes": "Lampiran undangan",
    "uploadDate": "2026-03-12T06:00:00Z",
    "uploadBy": "Budi Santoso",
    "downloadUrl": "/api/v1/attachments/501/download"
  }
}
```

---

## `GET /v1/mails/{mailId}/attachments`

List attachments surat.

**Response `200 OK`:**
```json
{
  "success": true,
  "data": [
    {
      "id": 501,
      "fileName": "lampiran.pdf",
      "fileExt": "pdf",
      "fileSize": 124000,
      "notes": "Lampiran undangan",
      "uploadDate": "2026-03-12T06:00:00Z",
      "uploadBy": "Budi Santoso",
      "downloadUrl": "/api/v1/attachments/501/download"
    }
  ]
}
```

---

## `POST /v1/archives/{archiveId}/attachments`

Upload attachment ke arsip. Sama format.

---

## `GET /v1/archives/{archiveId}/attachments`

List attachments arsip. Sama format.

---

## `GET /v1/attachments/{attachmentId}/download`

Download file attachment. Returns binary stream.

**Response Headers:**
```
Content-Type: application/pdf
Content-Disposition: attachment; filename="lampiran.pdf"
```

---

## `DELETE /v1/attachments/{attachmentId}`

Hapus attachment. Hanya bisa jika surat/arsip masih DRAFT dan user adalah creator.

**Response `204 No Content`**
